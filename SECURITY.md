# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch state.

## Reporting a Vulnerability

Please report vulnerabilities privately before opening a public issue.

Recommended path:

1. Open a GitHub Security Advisory draft for this repository, or
2. Contact the maintainer directly and include:
   - affected version/commit
   - reproduction steps
   - impact assessment
   - any suggested fix

You can expect an acknowledgment within 7 days.

## Scope Notes

This project is firmware/monitor code. Security reports that involve:

- unsafe monitor command behavior,
- memory corruption in command parsing,
- privilege-like bypasses in protected ranges, or
- malformed input causing unexpected control transfer

are in scope.
